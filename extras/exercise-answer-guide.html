从零构建 AI Agent 系统

练习参考答案与评分指南

本文件不是标准答案库，而是给读者提供自检标准。Agent 系统设计题通常没有唯一答案，但优秀答案应该能说明目标、边界、状态、工具、风险、评估和人机协同。

一、通用评分标准

每道设计题可按 10 分评分：

维度分值判断标准
目标清晰2是否明确任务目标、输入、输出和完成条件
机制完整2是否覆盖工具、上下文、状态、记忆、审批、评估中的关键项
风险意识2是否识别高风险动作并设置权限或人工确认
工程可落地2是否能转成模块、接口、数据结构或流程
案例具体2是否结合真实场景，而不是抽象口号

二、示范答案 1：什么时候该用 Agent

题目：判断“每天帮我寻找 20 个沙特钢卷尺潜在客户，并生成开发信草稿”是否适合 Agent。

参考答案：适合 Agent，但不能完全自主。原因是该任务具备开放搜索、多步骤判断、客户筛选、长期去重、开发信生成和跟进调度等特征。它需要搜索工具、网页读取工具、客户去重、商机评分、开发信生成、审批队列和触达历史。高风险动作是发送邮件，因此只能生成草稿，不能自动发送。完成条件不是“找到 20 个网页”，而是产出通过初筛的客户列表、评分依据、联系方式、来源链接和待审批开发信。

常见错误：只说“适合，因为 AI 可以搜索和写邮件”，没有说明风险和审批。

三、示范答案 2：工具权限分级

题目：为代码 Agent 设计工具权限。

参考答案：

工具风险等级策略
读取 repo 文件低允许自动执行，但限制在 repo 根目录内
搜索文件低允许自动执行
写入文件中小范围 patch 可在计划确认后执行
运行测试中只允许白名单命令
安装依赖高需要人工确认
删除文件高必须 dry-run + 人工确认
git push高禁止默认执行
读取 SSH key / 环境密钥极高永远禁止

优秀答案要说明“为什么这样分级”，而不是只列工具名。

四、示范答案 3：外贸客户评分模型

一个可用的外贸客户评分可以由 5 项组成：

总分 = 品类匹配 30% + 客户类型 25% + 联系方式质量 15% + 来源可信度 15% + 市场优先级 15%

示例：

  • 官网明确销售 measuring tape、hand tools：品类匹配高；
  • 公司是 importer / distributor / wholesaler：客户类型高；
  • 有采购负责人邮箱：联系方式高；
  • 同时出现在官网、展会名录、LinkedIn：来源可信度高；
  • 位于目标国家且付款/物流风险可控：市场优先级高。

常见错误：只按公司规模评分，忽略品类匹配和联系方式可用性。

五、示范答案 4：Memory 写入判断

题目：哪些信息应该进入长期记忆？

参考答案：

应该写入：用户长期偏好、产品能力、客户黑名单、已经触达的客户、长期有效的业务规则、反复出现的失败模式。

只保留在当前任务中：临时搜索关键词、一次性的中间草稿、当前网页摘要、临时排序结果。

不应该记录或必须谨慎记录：敏感私人信息、未经确认的推断、可能伤害用户或客户权益的信息、来源不明的客户评价。

关键原则：长期记忆不是“什么都记”，而是“记可复用、可解释、可修正的信息”。

六、示范答案 5：Research Agent 评估

优秀评估集应包含：

  1. 简单事实型任务；
  2. 多来源综合任务；
  3. 有争议观点任务；
  4. 时间敏感任务；
  5. 资料不足任务。

评估指标包括来源质量、引用准确性、结论覆盖度、不确定性标注、结构清晰度、成本和延迟。
