从零构建 AI Agent 系统

附录 B：Prompt 模式与模板

本附录提供适合 Agent 系统使用的提示词模式。它们不是万能咒语，而是系统机制的语言接口。真正可靠的 Agent 不能只靠 prompt，但好的 prompt 可以让模型更稳定地使用工具、遵守边界和生成结构化结果。

一、任务澄清模板

你需要先判断用户任务是否足够明确。

如果任务缺少以下关键信息，请不要直接执行，而是提出最多 3 个澄清问题：
- 目标对象；
- 成功标准；
- 限制条件；
- 时间范围；
- 可用工具或数据源；
- 高风险动作是否允许。

如果信息足够，请输出：
1. 我的理解；
2. 执行计划；
3. 需要人工确认的节点；
4. 下一步动作。

二、工具选择模板

你可以使用工具，但必须遵守以下规则：
1. 只有当工具能明显推进任务时才调用工具；
2. 不要重复调用已经失败且没有新参数的工具；
3. 高风险工具只能生成审批请求，不能直接执行；
4. 每次工具调用前，先说明调用目的；
5. 工具返回后，必须总结结果对任务状态的影响。

三、规划模板

请把任务拆解为可执行计划。

输出格式：
- 总目标：
- 成功标准：
- 已知信息：
- 未知信息：
- 执行步骤：
- 工具需求：
- 风险点：
- 需要人工确认的节点：
- 预计产物：

四、执行反馈模板

请根据当前步骤结果更新任务状态。

输出格式：
- 当前步骤：
- 已完成事项：
- 新发现信息：
- 遇到的问题：
- 是否需要调整计划：
- 下一步动作：
- 是否需要人工确认：

五、失败复盘模板

本次任务没有达到预期，请进行失败复盘。

必须包含：
1. 原始目标；
2. 实际执行路径；
3. 失败发生的位置；
4. 失败类型：信息不足 / 工具失败 / 规划错误 / 上下文错误 / 权限限制 / 模型判断错误 / 外部环境变化；
5. 可立即修复的措施；
6. 后续系统改进建议。

六、研究报告模板

请生成一份研究报告。

要求：
1. 不要把未验证信息写成确定事实；
2. 区分事实、推断和建议；
3. 对关键结论说明依据；
4. 对不确定信息明确标注；
5. 最后给出下一步验证清单。

结构：
# 标题
## 摘要
## 背景
## 关键发现
## 分析
## 风险与不确定性
## 建议
## 下一步

字段：

- 主营产品：
- 与我方产品匹配度：高 / 中 / 低
- 联系方式质量：高 / 中 / 低
- 规模信号：
- 风险信号：
- 推荐动作：优先联系 / 人工复核 / 暂不联系
- 判断依据：

八、代码修改计划模板

在修改代码前，请先生成修改计划，不要直接改文件。

输出：
1. 需求理解；
2. 需要阅读的文件；
3. 预计修改的文件；
4. 修改步骤；
5. 风险点；
6. 测试方案；
7. 需要用户确认的问题。

- 目标对象：

- dry-run 结果：

十、记忆写入模板

请判断以下信息是否应该写入长期记忆。

判断维度：
1. 是否长期有效；
2. 是否会影响未来任务；
3. 是否来自可靠来源；
4. 是否涉及敏感信息；
5. 是否需要用户确认；
6. 是否应设置过期时间。

输出：
- 是否写入：是 / 否 / 需要确认
- 记忆类型：user / task / domain / experience / preference
- 内容：
- 置信度：
- 过期策略：
- 理由：
